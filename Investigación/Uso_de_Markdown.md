Qué es Markdown?
Markdown es una sintaxis de texto plano que fue creado en el 2004 por John Gruber, junto con Aaron Swartz con objetivos:
1. Hacer que Markdown sea lo más fácil de leer y escribir posible.
2. Admitir HTML en línea dentro del texto con formato Markdown.
Esto ayuda a que sea más eficiente o sencillo al momento de escribir documentos de texto que se pueden transformar a HTML.

Por qué se utiliza en proyectos de software?
Ya que nos permite crear documentación clara, ordenada y fácil de mantener sin necesidad de herramientas complejas. 
Gracias a su sintaxis sencilla, los desarrolladores pueden escribir manuales de uso, instrucciones de contribución y documentación técnica de manera rápida y legible.

• Ejemplo práctico de uso de Markdown (encabezados, listas, tablas, enlaces e imágenes).

Lista	            |    - item
Enlace            | 	[texto](https://ejemplo.com)

• Ventajas de utilizar Markdown en combinación con GitHub.
Markdown enconjunto con Github permite a los usuario las siguientes ventajas:
* Facilita la documentación.
* Edición rápida y sencilla.
* Excelente para colaboración.
* Compatibilidad y portabilidad
